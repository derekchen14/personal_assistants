§ 01 / Premise
Why another framework
01

The ambiguity
bottleneck.

Diagnosis
Read time · 90s

Most "agent" frameworks live in the same loop: read input, fill a prompt with the tool list and history, parse a tool call, run, repeat. The loop works on demos because demo inputs are well-formed. Production inputs aren't.

Real users say "rework the marathon post" and mean one of six things. They say "end of day" and that means 5pm at a law firm and midnight at a startup. They say "make it pop" and the model has no shared context to ground it.

A typical agent guesses, commits, and hopes. It's not an intelligence problem — the model is plenty smart. It's that the cost of being wrong is hidden inside a confident answer, and the loop has no way to see it.

The last mile of agent design isn't tool-use. It's knowing when to stop and ask.

Typical agent

user › rework the marathon post

// model picks a tool, guesses, commits
agent › edit_post(id="three-hour-marathon",
            mode="polish")

agent › Here is a polished version…
Wrong post. Wrong scope. No way to notice.

An Assistant Factory agent

user › rework the marathon post

// state.entity has 2 candidates
// state.scope is missing
// uncertainty.level = "partial" + "specific"

hugo › Two posts about marathons —
       How I finally broke three hours
       or What marathon training taught
       me about discipline? Rework how —
       re-outline, re-voice, or polish?
One question. Two clarifications. Plan held.
§ 02 / Principles
What every assistant inherits
02

Three things every
assistant inherits.

Architecture
domain-agnostic
A
Belief, not transcript

What we think we're doing right now.

The agent carries an explicit object — current goal, entities in play, what's filled, what's missing — that survives between turns. Not a string of past messages re-read every call.

→ stable across long conversations
B
Memory, layered

What we remember, by horizon.

A short-horizon scratchpad for what we just learned, a per-account preference store for how you work, and slow-moving context about your team. Different reads, different writes.

→ "the usual tone" actually means something
C
Uncertainty, declared

What we don't know — at what level.

Calibrated and inspectable. Each turn the agent rates how unsure it is about each part of the request, and that rating decides whether to ask, confirm, or proceed.

→ asking one good question is a feature

Together these are why the agent doesn't reset on turn 7 — it resumes.

Ambiguity meter · this turn

Four levels of unsure. The agent rates each, and the rating chooses the response.

A · general Intent unclear "can you help"
B · partial Entity unresolved "the marathon post" → 2 match
C · specific Slot missing "rework" → which scope?
D · confirm Sign-off needed before publishing

Two declared ambiguities this turn → one targeted question that resolves both.

Why this is the unlock

Once uncertainty is a tracked object, every interesting agent property becomes implementable: multi-turn coherence, calibrated abstention, debuggable failures. They were unreachable when state lived inside a prompt.

§ 05 / Inside
Five questions, five answers
05

How an Assistant Factory
agent actually thinks.

Components · 5
Domain-agnostic

Each piece of the agent answers one question a non-engineer would actually ask. We picked the five that matter most — left out the plumbing.

A

World Model

"Does it know what my work is?"

A typed model of your domain — for Hugo: posts, sections, snippets, channels. The agent grounds requests against real things, not strings.

B

Dialogue State

"Does it know what we're doing now?"

The current goal, entities in play, what's filled and what's missing — held as a structured object that survives between turns.

C

Context Coordinator

"Does it remember this conversation?"

Conversation history, structured. Not a transcript blob — a coordinator that turns the back-and-forth into something the agent can index.

D

Memory Manager

"Does it remember me?"

Three horizons: a session scratchpad, account preferences, and slow-moving context about your team and your work. Different reads. Different writes.

E

Ambiguity Handler

"Does it know when to ask?"

Four levels of unsure — general, partial, specific, confirmation — declared per turn, with the response strategy bound to each level.

"Belief lives in the dialogue state. Memory is the manager. Uncertainty is its own first-class citizen. The model of you and your work is the world model. That's the system."
