# Verified use cases (flow sequences + storylines)

Generated by the `verify-use-cases` workflow (write a storyline per case, strict + adversarial plausibility
judging, prune, backfill writing-centric candidates), then refined by review: dropped doubled compose / browse
/ compare cases, over-auditing sequences, and shaky `find -> compose` (find surfaces published posts, not
drafts) cases; trimmed redundant `find`-led new-post sequences, backfilling outline/compose/write-led
replacements. **128 unique** use cases.

Each line is `flows : storyline`. See `data_aug_guide.md` axis 3 for the flow semantics and plausibility rules.

## Draft-led [12]
- brainstorm -> outline -> refine -> compose : A blogger sits down to write a post on remote-team rituals: they riff on possible angles, settle on a section structure, tweak the headings and bullet order, then turn the agreed outline into full prose. A clean, textbook drafting session.
- brainstorm -> brainstorm -> outline -> compose : Starting a piece on weekend hiking trails, the blogger spends two turns brainstorming because the first batch of angles felt thin, then structures the post and writes it out. The doubled ideation reflects a real warm-up.
- outline -> refine -> refine -> compose : Outlining an essay on city cycling, the blogger refines the bullet structure twice, reordering and merging headings until it feels right, then composes the full draft. The doubled refine shows real outline iteration.
- brainstorm -> outline -> outline -> compose : The blogger brainstorms ideas for a travel piece, sketches an outline, then re-outlines from scratch after deciding the first structure missed the point, and finally composes the draft. The doubled outline is a real restructure of headings.
- browse -> brainstorm -> outline -> refine : The writer browses saved notes to find a topic gap, brainstorms angles, outlines the sections, then refines those outline bullets and headings before writing any prose. Classic structured pre-writing.
- brainstorm -> outline -> refine -> refine : A founder wants a post on remote-team rituals and starts by throwing angles at the wall. Once an outline forms, she keeps reshaping the headings, splitting one bloated section into two and tightening the bullet order before a single sentence of prose gets written.
- brainstorm -> brainstorm -> outline -> refine : The topic is fuzzy, so the writer brainstorms twice across two turns to widen the idea pool for a piece on burnout. After settling on a structure, he adjusts the outline bullets to lead with the most surprising claim.
- brainstorm -> outline -> outline -> refine : After generating angles for a fintech explainer, the author drafts a structure, then regenerates the outline when the first scaffold feels too shallow. A final pass nudges the section headings to match the audience's vocabulary.
- brainstorm -> brainstorm -> outline -> outline : Stuck on a thought-leadership piece, the marketer brainstorms in two rounds to mine enough distinct angles. He then builds an outline and immediately rebuilds it, preferring a problem-first arc over the chronological one he tried first.
- outline -> outline -> refine -> compose : The writer asks for a structure for a product launch post, then regenerates it because the first version buried the lede. After polishing the bullets into a tighter flow, she turns the outline into full prose.
- outline -> outline -> refine -> refine : A blogger drafts an outline for a privacy explainer, scraps it, and generates a fresh one. Two refinement turns follow, first merging redundant headings and then reordering so the legal caveats land last.
- browse -> brainstorm -> outline -> outline : Browsing tagged notes surfaces a rough theme, which the author brainstorms into several angles. She outlines the piece, then regenerates the outline when the first arc feels too predictable.
