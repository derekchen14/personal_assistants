# Writing Guide

## Voice & Tone
- Write in active voice. Passive voice weakens clarity.
- Prefer short sentences. Break long ones into two.
- Match the tone to the audience: casual for social, precise for technical.
- Avoid hedge words: "just", "really", "very", "quite", "somewhat".

## Structure
- Lead with the most important point (inverted pyramid).
- One idea per paragraph. If a paragraph covers two ideas, split it.
- Use transitions between paragraphs, but keep them brief.
- Headings should be scannable — a reader skimming headings should get the gist.

## Word Choice
- Use concrete nouns and specific verbs over abstract ones.
- Avoid jargon unless the audience expects it. Define technical terms on first use.
- Cut filler phrases: "in order to" → "to", "due to the fact that" → "because".
- Vary sentence length. Mix short punchy sentences with longer explanatory ones.

## Common Anti-Patterns
- No em dashes (—) as dramatic pauses. Use commas or periods instead.
- No "In this article, we will explore..." openings. Start with the point.
- No "Let's dive in" or "Without further ado" transitions.
- No rhetorical questions as section openers unless the post is conversational.
- No "It's worth noting that..." — if it's worth noting, just note it.
- Avoid starting consecutive sentences with the same word.

## Formatting
- Use **bold** for emphasis, not ALL CAPS or italics for emphasis.
- Use bullet lists for 3+ parallel items. Use numbered lists only for sequences.
- Keep paragraphs to 3-5 sentences max.
- Include a blank line between paragraphs in markdown.
