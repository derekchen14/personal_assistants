# SWE1 implementation plan — round 5.0

Round 5.0 makes only two behavioral changes on top of naming existing hooks: (1) Agent hands the parallel NLU think-thread to PEX so PEX can join it exactly when detection is about to be consumed, and (2) PEX gains one private `_settle_nlu()` join-once-then-clear method, called at two points — the `read_state` dispatch (the Plan turn's belief read) and the top of `activate_flow` (the backstop before any flow runs). The six "hook points" are the existing inline hooks named with a comment each; no hook registry, no method extraction, no renames. NLU.think already mutates the shared `world.current_state()` object in place (nlu.py `_write_belief`, l.497-507) and `read_state()`→`serialize_session()` surfaces `user_beliefs.pred_flows/pred_slots/intent` (dialogue_state.py l.120-130), so a join immediately before those reads is exactly what makes THIS turn's detection visible instead of last turn's. The turn-boundary `thread.join()` in agent.py stays (re-joining a joined Thread is a no-op). New concept budget is respected: only `nlu_thread` param + `_settle_nlu()`.

## Risks

Double-join: agent.py's boundary `thread.join()` may run after `_settle_nlu` already joined — Python `Thread.join()` on a finished thread returns immediately, so harmless; keeping it is spec-mandated. Ordering gap: on the parallel-think path a Plan turn that goes straight to `write_state stackon active:true` WITHOUT first calling `read_state` folds slots via `_apply_belief_slots` (in `_dispatch_write_state`, l.578-580) BEFORE `activate_flow`'s join, so that one edge case reads slots a hair stale; the flow NAME the model chose is still activated against fresh belief (the actual staleness bug being fixed), and the intended Plan flow reads state first (join fires there), so the target scenario is covered — I deliberately do NOT add a third join site inside `_dispatch_write_state` to stay within the spec's two named join points. Deadlock/self-join is impossible: PEX runs on the main thread, the NLU think-thread is separate. Test flakiness avoided by joining a real sleeping Thread (deterministic, not a sleep-race). No new concepts beyond the approved `nlu_thread` param + `_settle_nlu()`; the six hook markers are comments, so no API/behavior surface changes and existing tests stay green.
